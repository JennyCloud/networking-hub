# Documentation in Networking

Proper documentation helps IT teams understand, maintain, and troubleshoot networks efficiently. In Network+, you are expected to understand different types of documentation and what they are used for.

## 1. Physical vs. Logical Diagrams

### Physical Network Diagram

A physical diagram shows the **actual physical layout** of the network.

It includes:
- Devices (routers, switches, firewalls, servers)
- Physical connections (cables)
- Rack locations
- Port connections

Purpose:
- Shows where devices are located
- Helps during hardware replacement or cabling issues
- Useful for onsite technicians

Think: *“Where is the device and what is plugged into what?”*

---

### Logical Network Diagram

A logical diagram shows how data flows through the network.

It includes:
- IP addresses
- Subnets
- VLANs
- Routing paths
- Network segments
- Firewall zones

Purpose:
- Shows traffic flow
- Helps troubleshoot routing and connectivity issues
- Used for planning network changes

Think: *“How does traffic move?”*

---

## 2. Rack Diagram

A rack diagram shows how equipment is mounted inside a server rack.

It includes:
- Rack units (U positions)
- Switches, routers, patch panels
- Servers
- UPS devices

Purpose:
- Helps with hardware organization
- Assists in planning space and power usage
- Speeds up hardware replacement

Example:
A 24-port switch might be mounted in Rack 3, U position 12.

---

## 3. Cable Map

A cable map documents how cables connect devices.

It includes:
- Cable types (Cat6, fiber, etc.)
- Port numbers
- Patch panel connections
- Wall jack mappings

Purpose:
- Troubleshoot connection issues
- Identify unused or mislabeled cables
- Track structured cabling

Without a cable map, troubleshooting becomes guesswork.

---

## 4. Network Diagram (General Term)

A network diagram can refer to either physical or logical diagrams.

It visually represents:
- Devices
- Connections
- Network segments

It is often used in documentation, presentations, and change planning.

---

## 5. Asset Inventory

An asset inventory is a list of all network-related equipment.

It includes:
- Device name
- Model
- Serial number
- IP address
- Location
- Warranty information
- Owner/department

Purpose:
- Track hardware lifecycle
- Manage replacements
- Support auditing and compliance

This prevents “mystery devices” in the network.

---

## 6. IPAM (IP Address Management)

IPAM is a system used to manage and track IP addresses.

It includes:
- Assigned IP addresses
- Available IP addresses
- Subnet tracking
- DHCP scopes
- DNS integration

Purpose:
- Prevent IP conflicts
- Keep address assignments organized
- Document subnet usage

Without IPAM, large networks quickly become chaotic.

---

## 7. SLA (Service Level Agreement)

An SLA defines service expectations between provider and customer.

It includes:
- Uptime guarantees (e.g., 99.9%)
- Response time expectations
- Resolution time targets
- Penalties for failure

Purpose:
- Set performance standards
- Define accountability
- Manage customer expectations

Example:
An SLA might guarantee 4-hour response time for critical issues.

---

## 8. Wireless Surveys

A wireless survey analyzes Wi-Fi performance and coverage.

It includes:
- Signal strength measurements
- Interference detection
- Access point placement
- Channel usage

Types:
- Passive survey (measure existing signals)
- Active survey (test performance while connected)
- Predictive survey (simulate coverage before installation)

Purpose:
- Optimize coverage
- Reduce dead zones
- Improve performance

Wireless surveys are essential before deploying enterprise Wi-Fi.

---

# Why Documentation Matters

Documentation reduces downtime, speeds up troubleshooting, and prevents configuration mistakes.

In networking, undocumented systems create risk.

Good documentation = predictable systems.
Poor documentation = mystery failures.
