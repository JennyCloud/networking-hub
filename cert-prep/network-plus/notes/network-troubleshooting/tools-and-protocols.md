## Tools and Protocols

### 1. Protocol Analyzers
- **Purpose:** Capture and analyze network traffic to troubleshoot and monitor network performance.
- **Examples:** Wireshark, tcpdump.
- **Use Cases:**
  - Identifying network bottlenecks.
  - Detecting unusual traffic or security issues.
  - Debugging network protocol problems.
- **Notes:** Often requires knowledge of protocols (TCP/IP, HTTP, DNS) to interpret captured packets effectively.

### 2. Command Line Tools
- **Purpose:** Built-in tools for network diagnostics and configuration.
- **Examples & Functions:**
  - `ping` – Tests connectivity between devices.
  - `tracert` / `traceroute` – Shows the path packets take to a destination.
  - `ipconfig` / `ifconfig` – Displays IP configuration information.
  - `nslookup` / `dig` – Queries DNS records.
  - `netstat` – Displays active connections and listening ports.
- **Use Cases:**
  - Quick troubleshooting of connectivity issues.
  - Verifying IP addresses and routes.
  - Checking open ports and active sessions.

### 3. Cable Testers
- **Purpose:** Verify physical network cables and connectors for proper wiring and continuity.
- **Types:**
  - Basic continuity testers – Check for open or short circuits.
  - Advanced testers – Measure signal quality, crosstalk, and cable length.
- **Use Cases:**
  - Identifying faulty cables.
  - Ensuring correct pinouts for twisted pair cables (Ethernet).
  - Validating fiber optic connections.

### 4. Wi-Fi Analyzers
- **Purpose:** Assess and troubleshoot wireless networks.
- **Examples:** NetSpot, Acrylic Wi-Fi, inSSIDer.
- **Functions:**
  - Detecting available Wi-Fi networks and signal strength.
  - Measuring channel usage and interference.
  - Identifying rogue access points.
- **Use Cases:**
  - Optimizing Wi-Fi coverage and performance.
  - Troubleshooting weak signal areas.
  - Security assessments of wireless networks.
