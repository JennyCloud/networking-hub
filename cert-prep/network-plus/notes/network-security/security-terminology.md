# Security Terminology

## 1. Risk
- **Definition:** The potential for loss or damage when a threat exploits a vulnerability.
- **Example:** A company storing sensitive data in a cloud server without proper encryption faces the **risk** of data theft.
- **Key Point:** Risk is a combination of **threat × vulnerability × impact**.

## 2. Vulnerability
- **Definition:** A weakness or flaw in a system, network, or process that could be exploited.
- **Example:** Unpatched software or weak passwords are common **vulnerabilities**.
- **Key Point:** Vulnerabilities are **internal weaknesses**, not attacks themselves.

## 3. Exploit
- **Definition:** A technique or tool used to take advantage of a vulnerability.
- **Example:** A hacker uses a known buffer overflow to gain unauthorized access to a server.
- **Key Point:** An exploit is the **action** that leverages a vulnerability.

## 4. Threat
- **Definition:** Anything that has the potential to cause harm by exploiting a vulnerability.
- **Example:** Malware, phishing attacks, or insider threats are all **threats**.
- **Key Point:** Threats are **external or internal forces** that can lead to security incidents.

## 5. CIA Triad
The foundation of information security, focusing on three core principles:

1. **Confidentiality**
   - Ensures information is accessible only to authorized individuals.
   - Example: Encrypting sensitive files so only authorized users can read them.

2. **Integrity**
   - Ensures information is accurate and unaltered.
   - Example: Using checksums to verify file integrity after transfer.

3. **Availability**
   - Ensures systems and data are accessible when needed.
   - Example: Implementing redundant servers to maintain uptime.

- **Key Point:** Security controls aim to protect all three components of the CIA triad simultaneously.
